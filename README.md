# Ground Zero Community - Configuration Repository

**Code Beyond Limits, Build Beyond Boundaries**

This repository (the `.github` special metadata repository) is the centralized home for community standards, guidelines, workflows, and default templates applied across all Ground Zero Community projects on GitHub.

## Repository Contents

* **[profile/README.md](profile/README.md)** – Customizes the organization's public landing page on GitHub.
* **[.github/CONTRIBUTING.md](.github/CONTRIBUTING.md)** – The central guide for community members outlining code contribution rules and workflow.
* **[docs/MARKDOWN_STYLE.md](docs/MARKDOWN_STYLE.md)** – Guidelines for writing and formatting consistent Markdown files.
* **[docs/CODE_REVIEW.md](docs/CODE_REVIEW.md)** – Guidelines for conducting constructive and quality code reviews.
* **[docs/README_TEMPLATE.md](docs/README_TEMPLATE.md)** – A boilerplate README template for bootstrapping new project repositories.
* **[docs/RELEASE_CHECKLIST.md](docs/RELEASE_CHECKLIST.md)** – A guide and checklist for project maintainers when releasing software.
* **[.github/CODE_OF_CONDUCT.md](.github/CODE_OF_CONDUCT.md)** – The standards for respectful behavior in community spaces.
* **[.github/SECURITY.md](.github/SECURITY.md)** – Process for reporting security vulnerabilities responsibly.
* **[.github/pull_request_template.md](.github/pull_request_template.md)** – Default template and checklists for Pull Requests.
* **[.github/ISSUE_TEMPLATE/](.github/ISSUE_TEMPLATE/)** – Templates for submitting bugs, feature requests, and documentation requests.
* **[package.json](package.json)** – Configuration for local testing tools and linting dependencies.
* **[scripts/check-links.js](scripts/check-links.js)** – Recursive cross-platform script to test documentation links locally.
* **[LICENSE](LICENSE)** – The permissive MIT license text.
